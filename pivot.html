pandas DataFrame.pivot — Q&A

Your questions about pivot, with explanations and examples.

Q: What does .pivot() do?

A: It reshapes a DataFrame from long → wide format. You specify index (rows), columns (new columns), and values (cell values). It does no aggregation — it fails if there are duplicate (index, column) pairs.

Q: Can I call .pivot() with no arguments?

A: No. You must provide at least columns. If you omit everything, pandas raises a TypeError because it cannot reshape without knowing the new column dimension.

Q: Is columns mandatory?

A: Yes. columns is required because it defines the set of new column headers. index and values are optional: if index is omitted, pandas keeps the existing index; if values is omitted, all remaining columns are pivoted (often creating MultiIndex columns).

Q: What does columns=... actually do?

A: The unique values in the column you pass to columns become new columns in the pivoted DataFrame. For example, if columns="year" and there are five unique years, you will get five new columns (or groups of columns if multiple values are pivoted).

Q: If every value in the columns column is unique, what happens?

A: Then you will have exactly as many new columns in the pivoted DataFrame as there are unique values in that column.

Q: What happens when I also provide index=...?

A: That column becomes the new row index and no longer appears as a regular column. Its unique values line up as the row labels, while the unique values of the columns argument spread across as the new columns.

Q: How is pivot different from pivot_table?

A: pivot is just reshaping and fails if duplicate index/column pairs exist. pivot_table handles duplicates by aggregating them with a function (mean by default). So use pivot when the data is guaranteed unique, and pivot_table when aggregation is needed.

Defaults recap: columns is required; index and values are optional. No aggregation is performed — that's what pivot_table is for.
