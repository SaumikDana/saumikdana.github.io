The ICA version of your system

Strip away brokers and databases, and the same structure becomes a classic interview-style registry task: create, update, close, query, and return sorted results. 

store = {}

def open_position(symbol, position):
    store[symbol] = {
        "open_position": position,
        "closed_positions": []
    }

def close_position(symbol, result):
    state = store.get(symbol, {
        "open_position": None,
        "closed_positions": []
    })

    if state["open_position"]:
        state["closed_positions"].append(result)
        state["open_position"] = None

    store[symbol] = state

def closed_positions_sorted():
    rows = []

    for symbol, state in store.items():
        for position in state.get("closed_positions", []):
            rows.append((symbol, position))

    return sorted(rows, key=lambda row: row[0])

Final takeaway

The impressive part of this code is not exotic Python. It is disciplined use of ordinary tools: dictionaries for records, lists for collections, string operations for parsing, guards for bad data, and sorting keys for clean output.

That is why drilling the “boring” layer pays off: it is the layer that actually carries the system.
